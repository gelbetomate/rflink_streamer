# RFLink Streamer

RFLink Streamer is a Home Assistant HACS custom integration that connects directly to a raw RFLink TCP stream, such as an ESPHome serial-to-TCP bridge, and creates native Home Assistant entities automatically when new RFLink device IDs appear.

## Features

- Pure TCP to native Home Assistant entities
- Automatic discovery of lights, switches, sensors, and binary sensors
- Automatic reconnect handling for transient TCP disconnects
- Bidirectional control for light and switch devices
- No MQTT broker, add-on, or external service required

## Installation

### HACS

1. Open HACS in Home Assistant.
2. Add this repository as a custom repository:
   - URL: `https://github.com/gelbetomate/rflink_streamer`
   - Category: `Integration`
3. Install **RFLink Streamer** from HACS.
4. Restart Home Assistant.

### Manual installation

1. Copy `/custom_components/rflink_streamer` into your Home Assistant `custom_components` directory.
2. Restart Home Assistant.

## Configuration

Add the integration from **Settings → Devices & Services → Add Integration** and choose **RFLink Streamer**.

Configuration fields:

- **Host**: IP address or hostname of the RFLink TCP stream
- **Port**: TCP port of the stream, default `6638`
- **Reconnect interval**: Seconds between reconnect attempts, default `10`

## Supported entity types

- `light`
- `switch`
- `sensor`
- `binary_sensor`

## Auto-discovery

When a new RFLink device ID is first seen on the TCP stream, RFLink Streamer creates the matching Home Assistant entity automatically:

- Lighting-style protocols default to `light`
- Known motor/relay style protocols become `switch`
- Weather and measurement payloads become `sensor`
- Doorbell, PIR, and alarm style payloads become `binary_sensor`

Entities keep stable unique IDs across restarts based on the parsed RFLink device ID.

## Sending commands back to RFLink

Lights and switches can send commands back over the same TCP connection:

- `ON`
- `OFF`
- `SET_LEVEL=<0-F>` for dimmable lights

## Example RFLink lines

Incoming RFLink messages:

```text
20;01;NewKaku;ID=c6c6c6;SWITCH=1;CMD=ON;
20;02;Oregon TempHygro;ID=1a2b;TEMP=0234;HUM=50;BAT=OK;
20;03;AB400D;ID=deadbeef;CMD=ON;
20;04;NewKaku;ID=c6c6c6;SWITCH=1;CMD=SET_LEVEL=15;
```

Outgoing commands generated by the integration:

```text
10;NewKaku;c6c6c6;1;ON;
10;NewKaku;c6c6c6;1;SET_LEVEL=15;
10;AB400D;deadbeef;OFF;
```

## Compatibility

This integration is designed for RFLink raw TCP streams and works well with an ESPHome device acting as a serial-to-TCP bridge in front of an RFLink gateway.
